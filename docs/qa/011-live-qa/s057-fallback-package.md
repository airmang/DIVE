# S-057 발표 폴백 패키지 — 2026-08-14 부산 (Windows 현장 장비)

GO 판정과 무관하게 발표는 생존해야 한다 (011 spec 테마 8). 이 문서는
run-of-show와 장애 대비 시나리오의 캐논이다. [준비물]은 체크리스트 —
완료 시 날짜를 기입한다.

## 데모 시나리오 (본선)

**전제**: 1차 티어 QA GO + Windows 설치본 smoke(S-055) 통과 시.

1. (30초) DIVE 소개 — 감독 하 AI 코딩: 학생이 작성자가 아니라 감독자.
2. (2분) 새 프로젝트 → PRD 인터뷰 — 정적 체크리스트 앱. 아키텍처 결정
   카드(형태→스택)에서 학생 확정 장면 강조.
3. (1분) 계획 생성 → 검토 카드. 기준이 "클릭하면 취소선" 같은 관찰 가능
   문장임을 보여줌.
4. (3분) step-1 실행 → 파일 편집 권한 카드(diff) 승인 → 미리보기 실제 조작
   → 근거 연결 → 승인. "AI가 했다고 끝이 아니라 학생이 검증한다" 메시지.
5. (30초) EventLog/export — 연구 데이터로서의 로깅(provenance 포함) 언급.

**데모 프로젝트/모델 고정**: 사전 승인된 로컬 프로젝트(`live-interactive-01`
계열) + Pi 검증 목록의 모델 1개를 리허설에서 고정하고 당일 변경 금지.
후보: 네이티브 `claude-sonnet-5`(범프 후) 1순위, `gpt-5.4-mini` 2순위 —
리허설 성적으로 확정.

## 폴백 사다리 (위에서 아래로)

| 단계 | 트리거 | 행동 |
|---|---|---|
| F0 본선 | 정상 | 위 시나리오 그대로 |
| F1 부분 라이브 | 신규 프로젝트 경로 불안 | 새 프로젝트 생략, 사전 승인 프로젝트의 step 실행~검증만 라이브 |
| F2 로컬 완성본 | 모델/네트워크 장애 | 완성된 로컬 결과물(생성 앱) 시연 + EventLog 뷰 |
| F3 녹화본 | 앱 자체 이상 | 전체 여정 녹화 재생 + 실기 화면은 정지 상태로 |

**전환 규칙**: 한 단계에서 30초 내 복구 실패 시 즉시 다음 단계로. 현장에서
디버깅하지 않는다.
