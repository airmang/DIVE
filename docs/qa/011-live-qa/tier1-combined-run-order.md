## 저니 B — S-050: 계획 게이트 (P0-01 해소 확인)

1. 새 프로젝트 (`qa-tier1-checklist`), 한국어 UI.
2. PRD: **정적 체크리스트 앱** — 2026-07-10 보고서 재현 조건. 기준 예:
   "완료한 항목을 클릭하면 취소선이 표시된다" / "할 일 3개를 추가하면 목록에
   3개 항목이 보인다". 아키텍처: 정적 페이지 + HTML/CSS/JS.
3. 계획 생성 → **PASS: 계획 초안 검토 화면 도달 (반복 차단 없음)**.
4. 회귀 확인(신규 프로젝트 `qa-tier1-junk`): 완료 기준에 "적당히 잘"만 넣고
   계획 생성 → **차단 + 한국어 이슈 문구 + "통과하는 예시" 블록** 표시.
   (스크린샷 필수 — 자기통과 예시 블록이 이번 라운드의 핵심 산출물)

## 저니 C — S-053: PRD 인터뷰 투명성 + provenance

1. `qa-tier1-checklist` PRD 인터뷰에서 상세 한국어 답변 1회(사용자·범위·
   비범위·제약·완료 기준 포함) 전송.
   - 반영되면: 필드 하이라이트 확인. **PASS.**
   - 반영 안 되면: 문구가 종전 한 줄이 아니라 **"구조화하지 못했습니다" +
     다시 구조화 버튼**인지, 버튼이 같은 답변으로 재시도해 수렴하는지 확인.
2. 새 프로젝트(`qa-tier1-manual`)에서 **모든 PRD 필드를 손으로만 입력**하고
   확정 직전 카드 확인 → **"AI가 대화를 정리한 요약"이 아니라 학생 작성
   중립 문구**가 떠야 한다 (P1-03 재현 조건). (스크린샷 필수)
3. 혼합 케이스(AI 패치 1회 적용 + 일부 필드 수동 수정) → 혼합 문구에 학생
   작성 필드 이름이 열거되는지 확인.

## 저니 D — S-052: 성공 후 허위 오류 0 (P1-01 해소 확인)

3. **PASS: stall timeout 오류 0회.** 오류가 뜨면 즉시 스크린샷 + 시각 기록.

## 저니 F — S-056: UX polish 확인 (신규)

1. **세션 시작 빈상태**: 새 step 세션 시작 직후 "세션을 시작해 대화를
   시작하세요"가 아니라 **"세션이 시작됐어요 — 첫 메시지를 보내면
   시작됩니다"** 계열 문구가 보이는지. (스크린샷)
2. **관찰 다중 연결**: step 검토에서 기준이 2개 이상일 때 관찰 기록 폼에
   **체크박스 목록 + "이번 관찰을 관련 기준 모두에 적용"** 버튼이 뜨는지;
   전체 적용 후 기록 1회로 여러 기준이 충족되는지. 체크 해제 시 기록 버튼
   비활성 확인.
3. **프로젝트 보관**: 사이드바에서 QA 프로젝트 하나를 보관(보관 아이콘) →
   접힌 **"보관됨 (N)"** 섹션으로 이동하는지, 펼쳐서 클릭하면 정상 열리는지,
   보관 해제가 되는지. (스크린샷)
