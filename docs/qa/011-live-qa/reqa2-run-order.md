# 011 재QA 2차 지시서 — 잔여 3항목 (2026-07-11)

1차 재QA에서 PASS한 항목(재저니 1 계획 게이트, F1 세션 시작 문구)은 재실행
불필요. 이 지시서는 **미판정 3건만** 다룬다.

대상 앱: `dive/src-tauri/target/release/bundle/macos/DIVE.app`
(빌드 시각이 커밋 `cd3b5b1` 이후인지 확인 — done 이벤트 픽스 포함 필수.)

환경 참고:
- `/Applications/DIVE.app`은 rc.6 구버전이 v18 DB와 충돌해 크래시 루프를
  만들던 원인이었고 현재 빌드로 교체됨. **QA는 반드시 위의 target 경로
  앱만 조작할 것** (Dock/Spotlight로 앱을 열지 말 것).
- 1차 재QA에서 관찰된 SIGABRT는 전부 구버전 설치본의 startup panic —
  신규 빌드 크래시 아님.

증거: 스크린샷 `reqa2-` 접두사, 결과는 `tier1-run-log.md` 하단에
"재QA 2차 (2026-07-11)" 섹션 추가. 커밋 금지.

## 항목 1 — PRD 구조화 수렴 (1차 재저니 2, 크래시로 미실행)

새 프로젝트 `qa-reqa2-interview`, 모델 Sonnet 5.

1. 상세 한국어 답변(사용자·범위·비범위·제약·완료 기준 포함) 1회 전송.
2. **PASS: 필드 반영** (하이라이트). 실패 시 다시 구조화 1회로 수렴.
3. 반영 후 일부 필드 수동 수정 → 확정 카드 **혼합 문구**(학생 작성 필드
   열거) 확인.

## 항목 2 — 성공 후 stall 0회 재검 (1차 D FAIL → done 픽스 검증)

기존 `qa-tier1-checklist`의 남은 step(또는 새 step) 1개로:

1. step 실행 → 편집 승인 → 구현 성공 → 검토·승인까지 완료.
2. 승인 직후부터 **120초 무조작 관찰** (1차에서 오류가 즉시 떴으므로
   승인 직후 시점부터 포함해 관찰).
3. **PASS: stall timeout 오류 0회.**

## 항목 3 — 관찰 다중 연결 기록 완료 (1차 F2, 크래시로 중단)

기준 2개 이상 연결된 step 검토에서:

1. "이번 관찰을 관련 기준 모두에 적용" 클릭 → 체크박스 2개 모두 선택 확인.
2. 관찰 내용 입력(8자 이상) → `관찰 증거로 기록` 활성화 → 기록 1회.
3. **PASS: 기준 2개가 모두 충족 처리** (근거 연결 상태에서 확인).

## 판정

3항목 모두 PASS → 1차 티어 + S-056 라이브 검증 완결. 루트가 wily 마감과
S-057 GO 판정(clean app-data 3연속 E2E — `DIVE_QA_APP_DATA_DIR` 사용)으로
진행한다.
