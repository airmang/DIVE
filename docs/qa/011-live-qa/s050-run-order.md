# 011 라이브 QA 작업 지시서 — S-050 계획 품질 게이트 (2026-07-11)

macOS computer-use 세션용. 대상 앱은 이미 빌드·실행되어 있음:
`~/DIVE-2/dive/src-tauri/target/release/bundle/macos/DIVE.app`
(꺼져 있으면 `open <경로>`로 실행. dev 모드 금지 — 릴리스 .app만 유효.)

## ⚠️ 선행 조건 (중요)

- **모델은 GPT-5.4 Mini 유지.** OpenRouter/네이티브 `claude-sonnet-5`를 선택하면
  안 됨 — S-051(P0-02, Pi sidecar `model not found`)은 아직 미수정이라 계획
  생성이 게이트와 무관하게 실패한다. 현재 선택 모델이 GPT-5.4 Mini인지 설정에서
  먼저 확인할 것.
- UI 언어는 한국어 상태로 진행.

## 저니 1 — QA 보고서 재현 (P0-01 해제 확인, 필수)

1. 새 프로젝트 생성 (이름 예: `qa-s050-checklist`, 경로는 기본값).
2. PRD 인터뷰/보드에서 **정적 체크리스트 앱**을 정의한다 — 2026-07-10 보고서와
   같은 조건: 목표에 "화면", "버튼" 같은 일반어가 들어가도 됨. 완료 기준 예:
   - "완료한 항목을 클릭하면 취소선이 표시된다"
   - "할 일 3개를 추가하면 목록에 3개 항목이 보인다"
3. PRD 확정 (아키텍처 결정 포함 — 정적 페이지 + HTML/CSS/JS 선택).
4. **계획 생성** 실행.

**PASS 기준**: 계획 초안이 생성되어 검토 화면에 도달한다 (반복 차단 없음).
스크린샷: 생성된 계획 화면 1장.

**차단될 경우** (그 자체로 FAIL이지만 추가 증거 수집):
- 복구 화면이 **한국어** 이슈 문구를 보여주는가? ("완료 기준이 너무 짧습니다…"
  등 — 영어 프로즈가 아니라)
- **"통과하는 예시"** 블록이 표시되는가?
- "짧은 JSON으로 다시 생성" 재시도 1~2회로 통과하는가? (예시가 프롬프트에
  주입되므로 이전과 달리 수렴해야 함)
- 각 상태 스크린샷.

## 저니 2 — 게이트가 여전히 정크를 막는지 (회귀 확인, 필수)

1. 새 프로젝트 (예: `qa-s050-junk`).
2. PRD 완료 기준에 의도적으로 "적당히 잘"만 넣고 확정 시도 → 계획 생성.

**PASS 기준**: 계획이 **차단되고**, 복구 화면에 한국어 이슈 문구 +
"통과하는 예시" 블록이 표시된다. 스크린샷 1장.
