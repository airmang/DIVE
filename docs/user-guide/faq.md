# DIVE FAQ

자주 묻는 질문. 답변을 못 찾았다면 [troubleshooting.md](./troubleshooting.md) → [GitHub Issues](https://github.com/coreelab/dive/issues) 순으로 확인.

## 설치 & 시작

### Q1. SmartScreen이 "PC를 보호했습니다"라고 막습니다. 악성 프로그램인가요?
**A.** 아니요. v1.0 시점에서는 EV 코드 서명 인증서가 아직 없어서 Windows가 "게시자 확인 불가"로 표시합니다. **추가 정보 → 실행**을 누르면 정상 진행됩니다. v1.1 이후 Azure Trusted Signing 도입 예정(roadmap: `docs/packaging-windows.md` §4).

### Q2. Windows 10인데 설치가 안 됩니다.
**A.** Windows 10 22H2 이상이어야 합니다. 그 이전 빌드(1903, 1909 등)는 WebView2 런타임이 누락되어 실패할 수 있습니다. Windows Update를 먼저 돌리거나, [`docs/troubleshooting.md`](./troubleshooting.md) §WebView2 섹션 참조.

### Q3. ARM64 맞는 인스톨러를 어떻게 고르나요?
**A.** 설정 → 시스템 → 정보 → "시스템 종류"에 **ARM 기반**이라 나오면 ARM64, **x64 기반**이라 나오면 x64입니다. Surface Pro X, Copilot+ PC는 대부분 ARM64. 일반 데스크톱·노트북은 거의 x64.

### Q4. macOS·Linux 버전은 없나요?
**A.** v1.0 범위는 Windows 우선(학교 보급률 기준). Tauri 구조상 macOS·Linux 빌드도 가능하지만 QA가 필요해 v1.1+에서 검토 예정 (명세 §14.3).

## 프로바이더 & AI

### Q5. ChatGPT Plus 구독으로 쓰려면?
**A.** 설정(Ctrl+,) → 프로바이더 → **ChatGPT로 로그인**. OAuth PKCE 흐름으로 API 키 없이 연결됩니다. 세션 토큰은 Windows Credential Manager에 안전 저장.

### Q6. Claude Pro 구독은 지원되나요?
**A.** 현재(v1.0) 미지원입니다. Anthropic의 공식 구독 API가 제한적이어서 API 키 방식만 지원. API 키는 [console.anthropic.com](https://console.anthropic.com)에서 $5 크레딧으로 충분합니다.

### Q7. OpenRouter는 무엇이 특별한가요?
**A.** 부모 API 키 하나로 학생별 **자식 키**를 자동 발급·폐기하는 **프로비저닝 키 API**를 지원합니다. 교사가 설정 → 프로비저닝에서 사용하면 학생은 API 키 노출 없이 쓰고, 수업 종료 후 일괄 폐기 가능.

### Q8. 같은 질문을 다른 모델에 다시 보내고 싶어요.
**A.** 사이드바 → 프로바이더 드롭다운에서 모델을 전환 후 같은 입력을 다시 보내면 됩니다. DIVE는 세션 컨텍스트를 유지하므로 이전 대화도 새 모델이 읽습니다.

## 단축키

### Q9. 전체 단축키 목록이 있나요?
**A.** 있습니다 (명세 §12.2 + ADR-034):

| 단축키 | 동작 | 폼 필드에서 발화되나? |
|---|---|---|
| `Ctrl+N` | 새 프로젝트 다이얼로그 | ✗ (타이핑 방해 방지) |
| `Ctrl+S` | 수동 체크포인트 | ✓ |
| `Ctrl+,` | 설정 | ✓ |
| `Ctrl+/` | 프롬프트 도우미 | ✓ |
| `Ctrl+E` | 슬라이드 인 패널 (코드/미리보기) 토글 | ✗ |
| `Ctrl+W` | 워크맵 접기/펼치기 | ✗ |
| `Tab / Shift+Tab` | 워크맵 카드 좌우 이동 | — |
| `Enter` | 카드 진입 | — |
| `A / R / E` | 권한 카드 승인·거부·편집 | — |
| `Ctrl+Shift+Enter` | 보내기 전 점검 (프롬프트 도우미) | ✓ |
| `Escape` | 다이얼로그·슬라이드 패널 닫기 | ✓ |

macOS 사용자는 Ctrl 대신 ⌘ 사용 가능 (`ctrlKey || metaKey` 모두 수용).

## 권한 카드 & 안전

### Q10. 권한 카드를 매번 보여주는 게 귀찮아요.
**A.** 의도된 설계입니다 (명세 §6.4 + §9.1). 입문자가 "AI가 뭘 하는지 확인하지 않고 다 허용"하는 패턴을 구조적으로 차단합니다. "항상 승인" 옵션은 같은 도구 + 같은 경로에서만 유효하므로 새 경로는 다시 묻습니다.

### Q11. 실수로 "항상 승인"을 눌렀어요.
**A.** 설정 → 권한 → 해당 항목 **철회** 버튼. 다음 호출 시 다시 물어봅니다.

### Q12. bash 도구가 실행되기까지 오래 걸려요.
**A.** 기본 타임아웃은 30초입니다. 지시에 `timeout_seconds=60` 같이 명시하거나, 설정 → 도구 → bash에서 기본값을 변경할 수 있습니다 (명세 §12.4).

## 체크포인트 & 되돌리기

### Q13. 체크포인트는 어디에 저장되나요?
**A.** 프로젝트 폴더의 `.dive/checkpoints/` 디렉터리. git 저장소라 `git log`로도 볼 수 있습니다. 앱이 사라져도 폴더만 있으면 복원 가능.

### Q14. 되돌리기를 했는데 이전 진행이 완전히 날아갔어요.
**A.** 복원 전 토스트에 "이전 상태를 별도 브랜치로 보존" 체크박스가 있습니다. 이 체크를 해제하면 덮어쓰기. 체크 유지가 기본값입니다.

### Q15. 다른 PC에서 같은 프로젝트를 이어서 하고 싶어요.
**A.** `.dive/` 폴더를 포함한 **전체 프로젝트 폴더**를 USB·클라우드로 복사 → 상대방 PC의 DIVE에서 "기존 프로젝트 열기"로 지정. 체크포인트 + 세션 히스토리 그대로 이전.

## 다국어 & 접근성

### Q16. 영어로 쓰고 싶어요.
**A.** 사이드바 하단 "한국어 / English" 토글. 즉시 반영. localStorage에 저장되어 다음 실행에도 유지. 앱 내 AI 프롬프트도 언어별로 분기됩니다.

### Q17. 스크린리더로 쓸 수 있나요?
**A.** NVDA·JAWS·내레이터와 동작합니다 (WCAG AA 준수). 사이드바·워크맵·채팅·권한 카드 모두 `aria-label` + 적절한 role 적용. 애니메이션은 `prefers-reduced-motion: reduce` 시 자동 억제. 모바일 보이스오버는 미지원 (데스크톱 전용).

### Q18. 고대비 모드가 지원되나요?
**A.** Windows 고대비 모드를 인식하지만 완벽한 매핑은 아직 아닙니다. v1.0 시점에는 파일럿 리포트(`docs/pilot-checklist.md`)에 수동 체크리스트로 포함. v1.1에서 공식 지원 예정.

## 데이터 & 프라이버시

### Q19. 내 코드가 AI 회사로 전송되나요?
**A.** 사용자가 지시에 포함한 **자연어 지시**와 **파일 컨텐츠(도구 호출로 AI가 읽은 경우)**만 해당 프로바이더로 전송됩니다. DIVE는 자체 서버로 코드를 보내지 않습니다(서버가 없음). ChatGPT·Anthropic·OpenAI 각 회사의 데이터 정책은 해당 이용약관을 따릅니다.

### Q20. 익명화 export는 무엇인가요?
**A.** 연구용 데이터 수집(명세 §6.7). 세션 로그를 **학생 이름·경로 등 식별자를 제거**한 JSONL로 내보냅니다. 설정 → 데이터 → **익명화 export**. 교사가 수업 후 일괄 분석할 때 사용하세요. 사용자 명시 동의 없이는 자동 전송되지 않습니다.

### Q21. 텔레메트리가 있나요?
**A.** 없습니다. v1.0 범위에서 사용 통계 수집 없음 (§14.5). 로컬 로그(`%APPDATA%\dive\logs\dive.log`)만 있으며 외부로 전송되지 않습니다.

## 기타

### Q22. 이 앱으로 생성한 코드는 제 것인가요?
**A.** 네. MIT 라이선스는 DIVE 앱 자체에만 적용됩니다. 앱을 사용해 **생성한 결과물**의 라이선스는 전적으로 사용자 재량입니다.

### Q23. 버그를 찾았어요.
**A.** [GitHub Issues](https://github.com/coreelab/dive/issues) 또는 연구진에게 직접 제보. 가능하면 `%APPDATA%\dive\logs\dive.log`를 첨부해주세요 (민감 정보는 자동 마스킹됩니다).

### Q24. 기여하고 싶어요.
**A.** PR 환영. 명세 변경은 사용자(연구진 총괄)만, 코드·문서·번역·버그 수정은 PR로. 기여 가이드는 README.md 참조.

### Q25. v1.0 이후 계획은?
**A.** [README.md 로드맵 섹션](../../README.md#로드맵) 참조. 교사 대시보드(v2.0)·macOS/Linux 지원(v1.1)·자동 업데이트(v1.1)가 후보.
