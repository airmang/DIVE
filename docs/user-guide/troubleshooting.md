# DIVE 트러블슈팅

증상 → 원인 → 해결 순으로 정리. 자주 발생하는 순서.

---

## 설치

### 증상: SmartScreen 경고 "게시자를 알 수 없음"

- **원인**: EV 코드 서명 미적용 (v1.0 시점 알려진 제약)
- **해결**: **추가 정보 → 실행** 클릭. 악성 프로그램이 아닙니다.
- **영구 대책**: Azure Trusted Signing 도입(v1.1 이후). 진행 상황은 [`docs/packaging-windows.md`](../packaging-windows.md) §4.

### 증상: 설치 중 "WebView2 런타임 설치 실패"

- **원인**: 오프라인 환경 + Windows 10 초기 빌드(WebView2 기본 미포함)
- **해결 A (온라인)**: 유선/Wi-Fi 연결 후 재설치
- **해결 B (오프라인)**: Microsoft 공식 [WebView2 런타임](https://developer.microsoft.com/microsoft-edge/webview2/)을 별도 다운로드 → 먼저 설치 → DIVE 재설치
- **참고**: Windows 11 / Windows 10 22H2 이상은 기본 포함

### 증상: ARM64 PC에서 x64 인스톨러 실행 시 "호환되지 않음"

- **원인**: 잘못된 아키텍처 인스톨러 선택
- **해결**: 설정 → 시스템 → 정보에서 "시스템 종류" 확인 → ARM64 인스톨러 다운로드 (`DIVE_<version>_arm64-setup.exe`)

## 첫 실행

### 증상: 앱을 실행했는데 창이 안 보임 / 까만 창

- **원인**: WebView2 초기화 실패 또는 GPU 드라이버 문제
- **해결**:
  1. `%APPDATA%\dive\logs\dive.log` 끝부분 확인 (예: "WebView2 init failed")
  2. WebView2 런타임 재설치 (위 증상 참조)
  3. 그래도 실패 시 GPU 드라이버 업데이트 또는 GitHub Issues에 로그 첨부 제보

### 증상: 온보딩에서 "폴더 선택"이 응답 없음

- **원인**: 프로젝트 폴더에 관리자 권한이 필요한 경로(예: `C:\Program Files\`) 지정
- **해결**: 사용자 홈(`%USERPROFILE%\Documents\dive-project` 같은 일반 폴더) 선택

## AI 프로바이더 연결

### 증상: ChatGPT OAuth가 브라우저에서 리디렉트가 안 되는 걸로 보임

- **원인**: 학교 방화벽이 `localhost:1455` 콜백 포트를 차단 + DIVE의 paste-mode fallback으로 넘어갔음
- **해결**:
  1. 브라우저 주소창에서 콜백 URL의 **`code=` 파라미터** 전체 복사
  2. DIVE 창의 "코드 직접 붙여넣기" 입력란에 붙여넣기 → 완료
  - 상세: 명세 §7.4 + `../internal/PHASE5_HANDOFF.md`

### 증상: "프로바이더 미연결"이 사라지지 않음

- **원인**: API 키 형식 오류 또는 만료
- **해결**:
  1. 설정 → 프로바이더 → 해당 프로바이더 → **연결 테스트** 버튼
  2. 실패 시 에러 메시지 확인 (예: "401 Unauthorized" → 키 만료)
  3. 콘솔에서 새 키 발급 후 교체

### 증상: "429 Rate Limited"가 반복됨

- **원인**: 크레딧 소진 또는 분당 요청 제한
- **해결**: 프로바이더 콘솔에서 잔여 크레딧 확인. OpenRouter는 무료 모델에서 자주 발생 → Claude/GPT 같은 유료 모델로 전환.

## DIVE 4단계 흐름

### 증상: 채팅창이 계속 잠겨 있음

- **원인**: 현재 단계의 게이트 조건 미충족 (의도된 동작)
- **해결**: 배너의 안내 메시지 확인
  - "먼저 워크맵에 카드를 추가하세요" → + 카드 추가
  - "카드에 지시를 작성해 주세요" → 카드 상세 패널에서 지시 입력
  - "검증 단계가 아닙니다" → 현재 카드가 verifying 상태여야 함

### 증상: AI 도구 호출이 "차단됨 (블록리스트)"

- **원인**: 차단된 명령(`rm -rf /`, `format C:` 같은 파괴적 명령)이 AI 응답에 포함됨
- **해결**: 정상 동작(명세 §9.2). 지시를 더 구체적으로 → "이 폴더만 정리"처럼 범위 한정.

### 증상: 카드 V 단계에서 "검증 실패"만 반복됨

- **원인**: 지시가 너무 모호해 AI가 의도를 놓침
- **해결**: I 단계로 회귀(거부) → 지시를 다음 기준으로 다시 작성:
  - **무엇을** 만드는지 (input 요소 1개)
  - **어떻게** (HTML + JS / React / …)
  - **확인 방법** (console.log / 화면 표시)
  - 프롬프트 도우미(Ctrl+/)가 좋은 예시를 템플릿으로 제공

### 증상: 한글 입력 중 Ctrl+, 등 단축키가 안 됨

- **원인**: IME(한글 입력) 세션이 Ctrl 키를 먼저 가로챔
- **해결**: 한영 키로 영문 입력 모드로 바꾼 후 단축키 사용, 또는 마우스로 해당 UI 클릭
- **항후**: v1.1에서 `e.isComposing` 체크로 IME 충돌 회피 검토 (ADR-034)

## 체크포인트 & 복원

### 증상: "체크포인트 저장 실패"

- **원인**: 프로젝트 폴더가 git 저장소가 아니거나, 디스크 공간 부족
- **해결**:
  1. `.dive/` 폴더 존재 확인
  2. 없으면 설정 → 프로젝트 → **체크포인트 재초기화**
  3. 디스크 공간 500MB 이상 확보

### 증상: 복원 후 코드가 예상과 다름

- **원인**: 복원 시점이 의도한 지점보다 이전일 수 있음
- **해결**: 사이드바 → 타임라인에서 각 체크포인트의 **라벨**과 **시점**을 한 번 더 확인 → 정확한 체크포인트 선택

### 증상: 이전 버전으로 되돌린 뒤 "database schema version ... is newer" 오류가 표시됨

- **원인**: 새 버전의 DIVE가 데이터베이스 구조를 앞으로 마이그레이션했습니다. 이전 앱은 더 새 schema를 안전하게 읽을 수 없어 시작을 거부합니다.
- **해결 A (권장)**: 다시 최신 DIVE 버전을 설치해 같은 데이터베이스를 엽니다.
- **해결 B (롤백)**:
  1. DIVE를 종료합니다.
  2. `%LOCALAPPDATA%\com.coreelab.dive\dive.db`를 다른 폴더에 복사해 보관합니다.
  3. `%LOCALAPPDATA%\com.coreelab.dive\backups\`에서 되돌릴 버전의 `dive-v<schema>-<timestamp>.db` 백업을 찾습니다.
  4. 기존 `dive.db` 이름을 `dive.db.newer`로 바꿉니다.
  5. 선택한 백업 파일을 `dive.db`로 복사합니다.
  6. 이전 버전의 DIVE를 실행합니다.
- **주의**: 백업 시점 이후의 세션, 메시지, 프로젝트 목록 변경은 이전 DB에 없을 수 있습니다. 프로젝트 폴더의 실제 파일은 앱 DB와 별개이므로 필요한 경우 별도로 백업하세요.

## 성능 / 메모리

### 증상: 앱이 느려지거나 멈춤 (500+ 메시지 세션)

- **원인**: 채팅 히스토리 누적 + 브라우저 엔진 메모리
- **해결**:
  1. Ctrl+S로 체크포인트 저장
  2. 새 세션 생성 (사이드바 → 세션 → + 새 세션)
  3. 이전 세션은 archived 상태로 보관 (자료는 보존)

### 증상: AI 응답이 15초 이상 걸림

- **원인**: 네트워크 지연 또는 프로바이더 과부하
- **해결**: 설정 → 네트워크 → 타임아웃 확인 (기본 60초). 30초 미만으로 줄이고 다른 프로바이더로 전환 옵션 활성화.

## 로그·진단

### 로그 위치

```
%LOCALAPPDATA%\com.coreelab.dive\logs\dive.log.YYYY-MM-DD  ← 일별 로그 (tracing INFO+)
```

API 키·OAuth 토큰·비밀번호 같은 민감 정보는 로그에 `[REDACTED_SECRET]`로 마스킹됩니다. 채팅 원문은 진단 로그에 남기지 않습니다.

### 진단 정보 수집 (이슈 제보용)

```powershell
# PowerShell
$env:LOCALAPPDATA + "\com.coreelab.dive\logs" | Set-Clipboard
```

위 경로를 탐색기에서 열고 가장 최근 로그 + `session.db`가 있는 프로젝트 폴더를 압축해 GitHub Issue에 첨부.

### 디버그 빌드 실행 (개발자용)

```powershell
cd dive
pnpm tauri:dev   # 자세한 tracing DEBUG 레벨 로그
```

## 알려진 제약 (v1.0)

- **EV 코드 서명 없음** → SmartScreen 경고 (Azure Trusted Signing 예정)
- **macOS·Linux 빌드 없음** → v1.1+ 계획
- **자동 업데이트 없음** → v1.1+ Tauri Updater 검토
- **모바일 지원 없음** → 데스크톱 전용 설계
- **IME 한글 입력 중 단축키 충돌** → v1.1 해결 예정
- **교사 대시보드 없음** → 익명화 export로 사후 분석 대체
- **실시간 Codex OAuth 검증** → 실 계정 기준 수동 스모크 테스트 필요 (§../internal/PHASE5_HANDOFF.md)
