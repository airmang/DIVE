# DIVE 튜토리얼 — 첫 30분 "할 일 앱 만들기"

처음 DIVE를 여는 학습자가 **30분 안에** 목표 정리 → 지시와 실행 → 변경 확인 → 검증과 승인 흐름을 한 번 완주하는 가이드. 명세 §3.1의 시나리오 A를 구체 지시로 풀어냈다.

**대상**: 고등학교 1학년 이상, HTML/CSS 기본을 아는 학생.
**소요 시간**: 30분 ~ 50분 (첫 실행 + 프로바이더 연결 포함).

---

## 준비물

- Windows 10 22H2 이상 또는 Windows 11 (x64 또는 ARM64)
- DIVE 인스톨러 (`DIVE_<version>_x64-setup.exe`) — [GitHub Releases](https://github.com/airmang/DIVE/releases/latest)
- **AI 프로바이더 계정 1개** (아래 중 하나)
  - **ChatGPT Plus/Pro 구독** — OAuth로 연결, 별도 API 키 불필요
  - Anthropic API 키 (콘솔에서 발급, $5 크레딧 필요)
  - OpenAI API 키
  - OpenRouter API 키 (자식 키 자동 관리)
- 인터넷 연결

---

## 0. 설치

1. 다운로드한 `.exe` 더블클릭
2. **SmartScreen 경고**가 뜨면 **추가 정보 → 실행** 클릭
   - 이유: v1.0 시점 코드 서명 미적용. 악성 프로그램이 아닙니다. (자세한 내용은 [FAQ](./faq.md))
3. NSIS 마법사 → **Korean** 선택 → Next · Next · Install
4. 설치 완료 후 바탕화면 "DIVE" 아이콘 더블클릭

---

## 1. 첫 실행 — 프로바이더 연결 (5분)

### 1-1. 온보딩

앱이 실행되면 온보딩 다이얼로그:

- **환영** 화면 → 다음
- **폴더 선택** → `바탕화면/dive-tutorial` 같은 빈 폴더를 "새 프로젝트" 버튼으로 지정
- **프로바이더 선택** — ChatGPT Plus가 있다면 **"ChatGPT로 로그인"** 추천

### 1-2. ChatGPT OAuth 예시

1. **ChatGPT로 로그인** 버튼 → 브라우저가 열림
2. 평소 쓰는 ChatGPT 계정으로 로그인
3. "DIVE 앱에 접근 권한 부여" → **허용**
4. 브라우저가 "DIVE로 돌아가세요" 안내 → DIVE 창으로 복귀
5. 사이드바 하단에 "ChatGPT · your-account-id" 표시되면 성공

> **참고**: API 키 방식을 선호하면 설정(Ctrl+,)에서 언제든 프로바이더 전환 가능.

## 2. 목표 나누기 (5분)

### 화면 레이아웃 이해

```
┌─────────────┬─────────────────────────────────────────┐
│  사이드바   │  채팅 영역 ("대화")                      │
│             │                                          │
│  프로젝트   │  [현재 단계 배너]                        │
│  세션       │                                          │
│  설정       │  메시지…                                 │
│             │                                          │
├─────────────┴─────────────────────────────────────────┤
│  워크맵 (하단 가로 띠) — 카드들이 가로로 나열         │
└───────────────────────────────────────────────────────┘
```

### 2-1. 첫 카드 추가

- 현재는 목표를 작은 작업으로 나누는 단계입니다. 채팅창은 잠겨 있습니다("먼저 워크맵에 카드를 추가하세요")
- 워크맵 왼쪽 **+ 카드 추가** 버튼 클릭
- 카드 제목: **"입력 폼"** 입력 → Enter

### 2-2. AI 도움으로 카드 3개 제안 받기

- 워크맵 **"✨ AI 제안"** 버튼 클릭
- 다이얼로그에 "간단한 할 일 앱을 만들고 싶어요" 입력 → 제안 받기
- AI가 다음 같은 3~4개 카드를 제안:
  - "입력 폼" (이미 추가함)
  - "할 일 목록 표시"
  - "완료 체크박스"
  - "삭제 버튼"
- **수락** 버튼으로 모두 워크맵에 추가

### 2-3. 카드 순서 정리

- 왼쪽부터 "입력 폼" → "할 일 목록 표시" → "완료 체크박스" → "삭제 버튼" 순서가 되는지 확인
- 순서가 다르면 드래그(v0.3 이후 지원)

## 3. 지시와 실행 (10분)

### 3-1. 첫 카드 진입

- 워크맵의 **"입력 폼"** 카드 클릭
- 채팅창 배너가 지시대로 작업을 진행하는 상태로 변경
- 카드 상세 패널이 오른쪽에 열림

### 3-2. 지시 작성

카드 상세 패널 하단 "지시" 입력란에:

```
input 요소와 button 요소 하나씩 포함한 간단한 입력 폼을 만들어줘.
button 클릭 시 input의 값을 console.log로 출력하게 해줘.
HTML + 순수 JavaScript로.
```

→ **지시 저장** 버튼.

> **잘 지시하는 요령** — 입력 폼을 예로:
>
> - "만들어줘" 같은 모호 표현 대신 "input과 button 하나씩"처럼 수치·구체 명시
> - 기술 스택 명시(HTML + JS / React / …)
> - 결과 확인 방법 명시(console.log / 화면에 표시 / …)

### 3-3. AI에게 실행 지시

채팅창 입력란에:

```
방금 저장한 지시대로 실행해줘.
```

→ Enter (또는 Ctrl+Enter).

### 3-4. 권한 카드 대응

AI가 **도구 호출**을 요청하면 **권한 카드**가 채팅 흐름에 나타남:

```
┌──────────────────────────────────────────────┐
│  ⚠️  파일 쓰기                               │
│  위험도: 보통                                 │
│  경로: dive-tutorial/index.html              │
│  ───────────────────────────────────────── │
│  [거부]  [편집]  [이번만 승인]  [항상 승인] │
└──────────────────────────────────────────────┘
```

- 내용을 **반드시 확인**한 후 **이번만 승인** 클릭 (`A` 단축키도 가능)
- AI가 `index.html` 생성 → 채팅에 결과 보고

### 3-5. 슬라이드 인 패널에서 코드 확인 (Ctrl+E)

- Ctrl+E → 오른쪽에서 슬라이드 인 패널이 열림
- **코드** 탭에서 방금 생성한 `index.html` 확인
- **미리보기** 탭에서 실제 input + button 렌더링 확인 → 클릭 테스트

## 4. 검증과 승인 (5분)

### 4-1. 검증 단계 진입

- 카드 상세 패널에서 **"검증 시작"** 버튼 클릭
- 배너가 검증 진행 중으로 변경
- DIVE가 검증 상태를 정리:
  - "지시와 일치: input과 button이 생성됨 ✓"
  - "기능 테스트: button 클릭 시 console에 값이 출력됨 ✓"

AI의 완료 보고는 주장으로 표시될 수 있습니다. 실제 승인 전에는 변경 사항, 미리보기, 실행 결과, 테스트 같은 검증 증거를 직접 확인하세요.

### 4-2. 사용자 최종 승인

AI 검증 결과를 **눈으로 확인**한 후:

- **[최종 승인]** 버튼 클릭 → 카드 상태가 검증 완료(녹색)
- 자동 체크포인트 저장 (토스트 "체크포인트 저장됨")

> **거부하고 싶다면** [거부] 버튼 → 지시 단계로 돌아가 지시를 수정해 재시도.

### 4-3. 카드 2-4 반복

남은 카드 3개도 같은 흐름으로:

1. 카드 클릭 → 지시 단계 진입
2. 지시 작성
3. "지시대로 실행해줘"
4. 권한 카드 승인
5. 슬라이드 인 패널로 결과 확인
6. 검증 시작 → 최종 승인

**팁**: 각 단계 전환 시점에 Ctrl+S로 수동 체크포인트 추가 가능. 문제 생기면 사이드바 "타임라인"에서 복원.

## 5. 확장과 마무리 (5분)

### 5-1. 모든 카드 검증 완료 확인

워크맵 하단 진행 표시가 모두 녹색이면 확장과 통합 옵션이 활성화됩니다.

### 5-2. 확장 단계 진입

- 마지막 검증 완료 카드 클릭 → "통합 점검" 옵션 활성화
- AI가 "모든 카드를 하나의 페이지로 통합했습니다"라고 보고
- 슬라이드 인 패널 **미리보기** 탭에서 완성된 앱 확인:
  - 입력 → 추가 → 목록에 표시 → 체크 → 삭제까지 모두 동작

### 5-3. 세션 종료

최종 승인 → 세션이 완료 상태로 바뀝니다. 프로젝트 폴더에:

```
dive-tutorial/
├── index.html
├── app.js
├── style.css
└── .dive/
    ├── session.db
    └── checkpoints/   ← git 히스토리
```

## 7. 자주 혼동하는 지점

### Q. "채팅창이 회색으로 잠겼어요"

**A.** 지금 단계의 필요한 조건이 충족되지 않았기 때문입니다. 배너에 "먼저 ~하세요"라고 이유가 표시됩니다. 이건 버그가 아니라 **DIVE의 의도된 동작**입니다.

### Q. "AI가 내 지시와 다른 걸 했어요"

**A.** 검증 단계에서 **거부** → 지시 단계로 돌아가 지시를 더 구체적으로 다시 쓰세요. 거부 후 다시 쓰는 것이 DIVE의 실제 작업 흐름입니다.

### Q. "체크포인트로 돌아갔는데 코드가 사라졌어요"

**A.** 체크포인트는 **git 기반 스냅샷**이므로 그 시점으로 완전히 되돌립니다. 되돌리기 전에 토스트에서 "이전 상태를 별도 브랜치로 보존" 옵션을 선택하면 잃지 않습니다.

축하합니다. DIVE의 첫 프로젝트 흐름을 한 번 완주했습니다.
