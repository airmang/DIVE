# 교사용 운영 매뉴얼

**DIVE 파일럿**을 25명 6차시로 운영하기 위한 선생님용 매뉴얼. 개발 경험이 없어도 따라올 수 있도록 작성되었습니다.

이 매뉴얼은 내부 파일럿 체크리스트(공개 저장소 미포함)와 짝입니다. 체크리스트는 "언제 무엇을 체크"이고, 이 매뉴얼은 "왜 이렇게 하는가 + 어떻게 대처하는가"입니다.

---

## 0. DIVE 철학 — 4단계 게이트

DIVE는 **"AI에게 무작정 맡기지 않는다"** 가 핵심입니다. 4단계:

| 단계 | 이름 | 학생이 하는 일 | 교사 개입 포인트 |
|---|---|---|---|
| D | Decompose | 만들 것을 **카드**로 분해 | 너무 큰 카드를 쪼개도록 유도 |
| I | Instruct | 각 카드에 **지시**를 작성 | 모호한 지시("예쁘게") → 구체적 지시로 ("빨강 × 18px 굵게") |
| V | Verify | AI 결과가 지시와 맞는지 확인 | 승인 전 diff 읽기를 강조 |
| E | Extend | 승인된 코드를 확장 | 새 카드 추가 타이밍 |

명세 §3~§4 참조. 각 게이트는 **코드로 강제**되므로 학생이 뛰어넘을 수 없습니다.

## 1. 파일럿 전 준비 (차시 1 직전)

### 1.1 환경

- 학교 PC 25대, Windows 10+ (Windows 11 권장)
- 안정된 무선/유선 네트워크
- 교사 PC 1대 + 태블릿(QR 코드 표시용)
- 프린트용 A4 25장 (QR 코드 출력 예비)

### 1.2 DIVE 설치

[`docs/windows-build-guide.md`](./windows-build-guide.md) 참조. 요약:

1. 선생님 PC에서 NSIS 인스톨러 다운로드 (개발팀이 제공하는 artifact)
2. 25대 학생 PC에 배포 (USB 권장)
3. 각 PC에서 SmartScreen 경고 → **추가 정보 → 실행** (선생님 지원 필요)

### 1.3 OpenRouter 계정

1. `https://openrouter.ai` 에서 메인 키 발급
2. $100 충전 (파일럿 6차시 예산 기준; 한 학생당 약 $1 / 차시)
3. 교사 앱에서 Provisioning 화면 → 25개 자식 키 발급 → QR 코드 받기

### 1.4 첫 차시 시나리오 선정

`docs/scenarios/session-01.md` ~ `session-06.md` 중 적절한 차시를 선택. 차시 번호가 학생 진도와 일치해야 합니다.

## 2. 차시 중 (50분 수업)

### 2.1 시작 5분 — 세팅

- 학생들이 자기 PC에 로그인하고 DIVE 실행
- 교사는 교실 앞에서 **시연**으로 "+ 새 프로젝트 → 새 세션 → 첫 카드 추가"까지 3분 보여주기
- 막힌 학생에게 빠르게 개인 지원

### 2.2 본 수업 35-40분 — 학생 주도 + 교사 순회

학생들은 혼자/짝으로 DIVE와 대화합니다. 교사는:

- **D 게이트 막힌 학생**: "만들 것을 작은 조각으로 나눠 봐" — AI 도움 받기 버튼 안내
- **I 게이트 막힌 학생**: "AI에게 무엇을 시킬지 구체적으로 적어 봐" — 예시 제공
- **V 게이트 고민하는 학생**: "결과가 지시랑 맞아? 그러면 승인, 아니면 거부" — 지시 수정 유도
- **권한 카드 보는 학생**: "diff에서 빨간 줄은 없어지는 거, 초록은 추가되는 거" — 차분히 읽기 권장

### 2.3 마무리 5-10분 — export + 회고

- 학생이 세션 export → JSONL 파일을 공유 폴더 또는 USB로 제출
- 교사가 3명 정도에게 짧은 질문:
  - "오늘 가장 헷갈렸던 순간?"
  - "AI가 엉뚱하게 했을 때 어떻게 했어?"

## 3. 자주 마주치는 상황

### 3.1 "AI가 이상한 답을 해요"

- 자연스러운 일. AI는 완벽하지 않습니다.
- 해결법: 채팅에 "다시 해줘" 또는 **Cancel** 버튼으로 중단 후 지시를 더 구체적으로
- 3회 연속 실패하면 다른 프로바이더로 전환 (설정 화면에서 교체)

### 3.2 "체크포인트로 돌아가고 싶어요"

- 슬라이드 인 하단 타임라인 → 원하는 점 호버 → 복원
- 확인 다이얼로그 표시: "현재 상태가 '복원 직전' 체크포인트로 자동 저장됩니다" → **복원하기**
- 이전 상태로 되돌아가도 **언제든 앞으로 돌아올 수 있음**

### 3.3 "실수로 파일을 지웠어요"

- 체크포인트 타임라인에서 직전 auto 체크포인트로 복원
- 복원 직전 자동 저장 덕에 "실수 복원" 자체도 되돌릴 수 있음

### 3.4 "권한 카드에서 뭘 승인해야 할지 모르겠어요"

- Safe (파란 테두리): 자동 승인됨. 신경 쓸 필요 없음
- Warn (노란 테두리): 파일 쓰기. **diff 확인 후 승인** — 빨강=지울 줄, 초록=추가할 줄
- Danger (빨간 테두리): 위험한 명령. **반드시 내용 읽고** 승인 또는 거부

### 3.5 "API 키가 안 돼요"

- OpenRouter 대시보드에서 해당 자식 키 상태 확인
- 한도 소진이면 교사가 한도 증가
- 완전히 죽은 키면 교사가 재발급 → QR 코드 다시 전달

## 4. 차시 간 준비

매 차시 후:

- [ ] OpenRouter 자식 키 25개 **일괄 폐기** (label prefix 기준)
- [ ] 다음 차시용 키 25개 재발급 (label prefix는 달라야 함: `class-tue-2-` 등)
- [ ] 수집된 JSONL 파일을 `dive-pilot-data/{YYYY-MM}/` 폴더에 정리
- [ ] 내부 파일럿 벤치마크 양식(공개 저장소 미포함)에 오늘 차시 데이터 기록
