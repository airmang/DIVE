# 차시 03 — JavaScript 이벤트 + 검증 실패 → 지시 수정

**학습 목표**: `<button>` 클릭 이벤트를 JS로 처리. AI 결과가 지시와 다를 때 **검증을 거부하고 지시를 수정**하는 경험.

**전제**: 차시 02 종료 후 HTML + CSS가 갖춰진 상태.

---

## 수업 흐름

### 0-5분: 복습

- 권한 카드 diff 읽기 1분 퀴즈 (화이트보드에 가짜 diff 표시 → 학생들에게 "승인? 거부?")

### 5-15분: D 단계 — JS 카드 2개

- 카드 A: 버튼 `<button id="greet">인사하기</button>` 추가
- 카드 B: script.js 생성 + 클릭 시 "안녕!" alert

### 15-35분: I → V 순환 — **검증 실패 체험**

**카드 A**: 지시 "body 안에 `<button id='greet'>인사하기</button>` 추가. 다른 변경 금지."
- AI가 `<script>` 태그까지 추가하려 하면 → **권한 카드에서 거부**
- 지시 재강조 → 다시 시도 → verified

**카드 B — 핵심 학습**: 지시 "script.js 생성 + 버튼 클릭 시 window.alert('안녕!')"
- AI가 `confirm(...)` 같은 다른 함수를 쓰면 → 검증 단계에서 **거부**
- 교사 유도: "결과가 지시랑 맞아? 안 맞으면 거부. 지시 수정해서 다시."
- 지시 수정 "**반드시 window.alert** 함수 사용" → 다시 시도 → verified

### 35-45분: 체크포인트 복원 시연

교사 시연:
1. 카드 B verified 완료 상태의 학생 PC 하나 선정
2. 슬라이드 인 → 타임라인 → **직전 체크포인트** (V 통과 자동 생성) 클릭
3. 복원 확인 다이얼로그 → **"복원 직전 자동 저장됨"** 강조 → 복원
4. 파일 원복 확인
5. 다시 앞으로 → 최신 체크포인트로 복원 → "되돌리기도 되돌릴 수 있다" 강조

### 45-50분: Export + 회고

---

## 교사 개입 포인트

### 검증 실패 유도

학생이 "AI가 대충 한 거 승인"하는 경향 방지:
- 교사: "결과 코드 열어봐. 지시랑 정확히 같아?"
- 학생: "비슷한 것 같은데..."
- 교사: "**비슷한** 거 말고 **정확히 같은** 거여야 해. 다르면 거부."

### "왜 거부해요? 된 것 같은데?"

- 교사: "지금은 맞아도, AI가 의도 이해 못한 채 우연히 됐으면 다음에 고장나. 지시 강화."

### 복원 시연의 효과

체크포인트 복원을 학생들이 **안전망**으로 인식하면 이후 차시에서 실수에 대한 두려움 감소.

## 예상 문제

- **JS alert이 브라우저에서 안 뜸**: 팝업 차단 설정 확인 → 주소창 왼쪽 아이콘에서 허용
- **버튼이 눌러도 반응 없음**: script.js가 HTML에 `<script src="script.js"></script>`로 연결 안 됨 → 카드 A 지시 재확인
- **학생이 검증 거부를 무서워함**: 거부해도 "다시 시도" 가능하고, 체크포인트로 전 상태 복원 가능 → 안심

## 차시 종료 체크

- [ ] 학생 50%+ 카드 2개 모두 verified (어려워지는 차시)
- [ ] 검증 거부 후 재시도한 학생 존재
- [ ] 체크포인트 복원 1회 이상 수행한 학생 존재
