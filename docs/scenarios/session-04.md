# 차시 04 — 상태 관리 (간단한 카운터) + 체크포인트 복원 연습

**학습 목표**: JavaScript 변수로 상태 관리. 의도적으로 실수한 후 체크포인트로 복원.

**전제**: 차시 03에서 JS 이벤트까지 완료.

---

## 수업 흐름

### 0-5분: 복습 + 오늘 목표

- "오늘은 일부러 실수해보고 되돌리기까지 해볼 거야"
- 카운터 예시 화이트보드: [−] 0 [+]

### 5-15분: D 단계 — 카드 3개

- 카드 A: HTML에 카운터 div + +/- 버튼 추가
- 카드 B: JS에 `let count = 0` + 업데이트 함수
- 카드 C: 클릭 이벤트 바인딩

### 15-35분: I → V 순환 × 3

**카드 A**: 지시 "`<div id='counter-display'>0</div>`, `<button id='inc'>+</button>`, `<button id='dec'>-</button>` 이 순서대로 body에 추가"
- verified → 자동 체크포인트

**카드 B**: 지시 "script.js에 let count = 0 + function updateCounter() { document.getElementById('counter-display').textContent = count; }"
- verified → 자동 체크포인트

**카드 C**: 지시 "inc 버튼 클릭 시 count++ + updateCounter(), dec 버튼 클릭 시 count-- + updateCounter()"
- verified → 자동 체크포인트

### 35-45분: 의도적 실수 + 복원 연습

**교사 유도**:
1. "이제 일부러 부수자. 채팅에 '카운터 삭제해줘' 라고 입력"
2. AI가 파일 수정 → 권한 카드 → **승인** (일부러)
3. 카운터가 사라짐
4. "이제 되돌릴 거야. 타임라인에서 카드 C 직후 체크포인트 클릭"
5. 복원 확인 다이얼로그 → 확인 → 원상복구
6. "선생님이 어떤 학생 실수해도 복원 가능하다는 걸 보여줬어"

**학생이 직접 실험**:
- 각자 다른 카드로 돌아가보기 (카드 A 직후 → 카운터 없는 상태 보기)
- 다시 최신으로 복원

### 45-50분: Export

## 교사 개입 포인트

### 체크포인트 종류 이해

학생들에게 타임라인 색 설명:
- 회색 테두리 (init): 초기
- 초록 (auto): V 통과 시 자동
- 보라 (manual): Ctrl+S로 직접 저장
- 파스텔 보라 + glow (current): "지금 여기"

### "실수해도 괜찮다"는 메시지

이번 차시는 **심리적 안전망** 조성이 목적:
- "체크포인트 = 게임 세이브"
- "실수해도 2단계 전으로 돌아갈 수 있어"
- 학생이 실험적으로 이상한 지시를 시도해 보도록 유도

## 예상 문제

- **카운터가 업데이트 안 됨**: `updateCounter()` 호출 누락 → 카드 C 지시 재확인
- **복원 후 혼란**: "앞으로 돌아가는 법?" → 최신 체크포인트 재클릭
- **체크포인트 너무 많음**: 수업 동안 각 학생 최대 6-10개 자동 생성 → 타임라인 가로 스크롤로 충분
