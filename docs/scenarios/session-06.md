# 차시 06 — 종합 프로젝트 + 발표

**학습 목표**: 5차시 동안 배운 내용(HTML + CSS + JS + 이벤트 + API + 에러 처리)을 **자유 주제**로 통합. 학생이 직접 DIVE로 프로젝트 하나 완성.

**전제**: 차시 01-05 완료. 학생이 D/I/V/E 사이클 독립 수행 가능.

---

## 수업 흐름

### 0-5분: 프로젝트 주제 제시

학생 자유 선택 (교사가 예시 3-4개 화이트보드):
- 자기소개 페이지 (이름 + 취미 + 사진)
- 할 일 목록 (추가 / 삭제 / localStorage 저장)
- 영어 단어 퀴즈 (API 또는 내장 배열)
- 오늘의 기분 기록 (이모지 + 메모)

### 5-15분: 프로젝트 시작

- 학생 각자 "+ 새 프로젝트" → 주제 기반 이름
- D 단계: AI 도움 받기로 4-6개 카드 초안 받기
- 교사 순회하며 카드 구성 검토 (너무 크면 쪼개도록)

### 15-40분: 자율 작업

학생들이 D/I/V/E 순환 독립 수행. 교사는:

- 막힌 학생 1:1 지원
- 체크포인트 복원 안내
- 권한 카드 diff 읽기 리마인드

**중간 체크** (25분 시점): "지금까지 verified 카드 몇 개야?"
- 0-1개: 카드 크기 너무 큼. 쪼개기
- 2-4개: 정상
- 5개 이상: 빠름. 추가 기능 (E 단계) 도전

### 40-48분: 짧은 발표

학생 5명 랜덤 지명:
- 주제 1줄 소개
- 화면 공유 또는 브라우저 열어서 보여주기
- "가장 어려웠던 카드?"

### 48-50분: 최종 Export + 설문

- Export JSONL 마지막으로
- **간단 설문** (Google Form 또는 종이):
  - 1. DIVE 전체 경험 1-5 점
  - 2. 가장 유익했던 단계 (D/I/V/E 중)
  - 3. 개선 원하는 부분 1가지

## 교사 개입 포인트

### 스스로 해결하도록 유도

- 직접 답 주지 말고 "지시를 어떻게 더 구체적으로?" 질문
- 체크포인트 복원이 있어 실수해도 안전함을 다시 강조

### 성취 축하

- verified 카드 5개 이상 학생 → 박수
- 발표자 → 박수
- "오늘 너희가 AI랑 일하는 법을 배웠어" 메시지

## 예상 문제

- **주제가 너무 큼**: "게임 만들기" → 너무 광범위 → "1초마다 숫자 +1 되는 페이지"로 축소 유도
- **시간 부족**: 50분에 못 끝내는 학생 → "오늘 여기까지 하고, 집에서 이어가도 돼" (DIVE는 로컬 저장)
- **발표 시 버그**: 당황하지 말고 "체크포인트로 복원" 시연 → 오히려 강력한 시연

## 차시 종료 체크

- [ ] 학생 70%+ 프로젝트 최소 1개 카드 verified 상태
- [ ] 발표자 5명 결과물 공유 완료
- [ ] Export JSONL 25개 최종 수집
- [ ] 설문 응답 20+ 수집

## 파일럿 종료 처리

이 차시가 6차시 파일럿의 **마지막**이면:

- [ ] OpenRouter 자식 키 25개 **최종 일괄 폐기**
- [ ] OpenRouter 메인 키 한도 확인 → 남은 크레딧 정리
- [ ] 모든 JSONL 파일을 `dive-pilot-data/final/`로 백업
- [ ] 내부 파일럿 벤치마크 양식(공개 저장소 미포함)에 전체 6차시 요약 + 교사 회고 작성
- [ ] `docs/pilot-feedback.md` 에 학생 설문 결과 집계
