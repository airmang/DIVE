# 차시 01 — DIVE 소개 + HTML 기초

**학습 목표**: 학생이 DIVE의 D/I/V/E 4단계를 1회 체험하고, 첫 HTML 페이지를 AI와 함께 만든다.

**소요 시간**: 50분

**준비물**:
- 학생용 DIVE 설치 완료
- OpenRouter 자식 키 25개 QR 코드
- 교사용 시연 프로젝트

---

## 수업 흐름

### 0-5분: 인사 + 시연

교사가 교실 앞에서 DIVE 실행 → 온보딩 → 프로젝트 생성 → 세션 시작까지 1회 시연. 화이트보드에 D/I/V/E 4 글자 크게 적기.

### 5-10분: 학생 DIVE 실행 + 프로바이더 연결

- 학생: 바탕화면 DIVE 실행 → QR 스캔 또는 키 입력 → 연결
- 교사: 막힌 학생 개인 지원

### 10-15분: 프로젝트 + 세션 생성

학생 각자:
1. "+ 새 프로젝트" — 이름 "첫 웹페이지", 경로 `C:/Users/{이름}/Desktop/dive-01`
2. "+ 새 세션" — 자동 세션 생성

### 15-25분: D 단계 — 카드 3개 자동 생성

학생 각자:
1. "AI 도움 받기" 버튼 클릭
2. 입력: "인사말이 나오는 간단한 HTML 페이지 만들어줘"
3. AI가 4개 제안 → 3개 선택 후 수락 (예: "제목 추가", "인사말 문단 추가", "이름 입력 칸 추가")

**예상 카드 구성**:
- 카드 1: HTML 기본 구조 (html/head/body)
- 카드 2: 제목(`<h1>안녕하세요!</h1>`)
- 카드 3: 이름 입력 칸(`<input>`)

### 25-40분: I → V 첫 순환 (카드 1 집중)

첫 카드만 I → V 돌리기:

1. 카드 1 클릭 → 상세 패널
2. **지시 작성**: "파일 이름은 index.html, doctype + html + head + body 태그만 최소"
3. 채팅창에 동일 내용 입력
4. AI가 write_file 호출 → **권한 카드** 팝업
5. 학생이 diff 확인 → **승인**
6. 파일 생성 → **[검증 요청]**
7. V 상태 → AI가 자체 검증 → **[승인]** → ✅ verified

**교사 강조 포인트**:
- 권한 카드의 diff 읽는 법 (+ 초록 / - 빨강)
- 검증 통과 후 자동 체크포인트 생성 (타임라인 초록 점)

### 40-45분: 카드 2, 3 빠른 반복

학생 각자 카드 2, 3도 동일 패턴으로 처리. 교사는 순회하며 지원.

### 45-50분: Export + 회고

1. 세션 export → JSONL 파일 저장
2. 교사 3명 질문:
   - "가장 헷갈렸던 순간?"
   - "AI가 엉뚱하게 할 때 어떻게 했어?"

---

## 교사 개입 포인트

- **D 게이트**: AI 도움 받기를 모르는 학생이 많음 → 3명 이상 멈추면 칠판에 재시연
- **I 게이트**: 지시가 너무 짧으면 ("그냥 해줘") AI가 임의로 해석 → "파일 이름은 X, 태그는 Y만"처럼 구체화 유도
- **V 게이트**: 승인 전 "결과 파일 직접 열어봐" 강조 (슬라이드 인 → 코드 탭)

## 예상 문제

- **권한 카드 팝업 무시**: 학생이 Enter 연타로 자동 승인 → 권한 카드에서 diff를 "한 줄이라도 읽기" 약속
- **카드 3개 다 못 끝냄**: 괜찮음. 첫 차시 목표는 1회 순환 체험. 남은 카드는 다음 차시 이어서
- **AI가 의도와 다른 파일 만듦**: 검증 단계에서 거부 → 지시 더 구체화 → 다시 시도

## 차시 종료 체크

- [ ] 25명 모두 프로젝트 1개 + 세션 1개 이상 생성
- [ ] 학생 80% 이상 카드 1개 이상 verified 도달
- [ ] 네트워크 오류 5회 이하
- [ ] Export JSONL 25개 수집
