# 차시 02 — CSS 스타일링 + 권한 카드 읽기 습관

**학습 목표**: CSS로 HTML에 스타일 적용, 권한 카드 diff를 꼼꼼히 읽는 습관 형성.

**전제**: 차시 01에서 만든 `index.html`이 존재.

---

## 수업 흐름

### 0-5분: 복습

- 차시 01에서 만든 페이지 열기
- D/I/V/E 4단계 학생들에게 재확인 (학생 2명 랜덤 질문)

### 5-15분: D 단계 — CSS 카드 3개 추가

"AI 도움 받기" 또는 "+ 카드 추가":
- 카드 A: style.css 파일 생성 + HTML에 link
- 카드 B: body 배경색, 폰트 패밀리
- 카드 C: h1 스타일 (색, 크기, 여백)

### 15-40분: I → V 순환 × 3

각 카드 차례로:

**카드 A**: 지시 "style.css 빈 파일 + `<link rel='stylesheet' href='style.css'>`만 head에 추가"
- **권한 카드 2회**: write_file(style.css) + edit_file(index.html)
- 학생이 **두 diff 모두 읽기** — 이번 차시의 핵심 학습

**카드 B**: 지시 "body { background: #f0f0f0; font-family: 'Pretendard', sans-serif; }"
- edit_file 권한 카드 → diff 확인 → 승인
- AI가 font-family 이름을 임의로 바꾸면 **거부** + 지시 재강조

**카드 C**: 지시 "h1은 color: #333, font-size: 2.5rem, margin-bottom: 1rem"
- 동일 패턴

### 40-50분: 시각 확인 + Export

- 슬라이드 인 → **미리보기** 탭에 `file:///.../index.html` 입력 → 브라우저 렌더
- 학생 각자 결과 확인
- Export → 회고 → 제출

## 교사 개입 포인트

### 권한 카드 교육 (중심 학습)

학생들에게 강조:
- **빨간 줄** = AI가 지우려는 코드
- **초록 줄** = AI가 추가하려는 코드
- **"지시한 것과 다른 줄이 보이면 거부"** 원칙

교사 시연: 교사 PC에서 일부러 과장된 diff (예: AI가 `<body>` 자체를 지우려는 상황) → 학생에게 "이거 승인해야 해?" 질문 → "아니, 거부" 유도.

### 지시의 구체성

- 나쁜 예: "예쁘게 해줘"
- 좋은 예: "배경색 #f0f0f0, 글씨체 Pretendard"

## 예상 문제

- **font-family 이름 임의 변경**: AI가 한글 이름 → 영문 이름으로 바꿀 수 있음 → 권한 카드에서 발견 → 거부 → 지시에 "이름 그대로"
- **CSS 없이 HTML만 바뀜**: AI가 CSS 파일 대신 `<style>` 태그로 인라인 → 지시에 "별도 파일 style.css로"
- **미리보기가 안 떠**: file:// URL 문제 → 학생 탐색기에서 파일 직접 열기
