# 작업 31: 다국어 (한국어/영어 리소스)

## 컨텍스트
명세 §2.5, §12.3에 따라 한국어와 영어를 동등 지원. 한국어가 기본, 영어 토글. AI 시스템 프롬프트도 언어별로 분기. 그동안 ko 텍스트만 하드코딩한 모든 사용자 노출 텍스트를 i18n 리소스로 분리하는 큰 작업.

## 이번 작업 범위
- `react-i18next` 통합
- 한국어(ko) / 영어(en) 두 리소스 파일
- 모든 사용자 노출 텍스트를 키-값으로 분리
- 첫 실행 시 OS 언어 감지 → 자동 선택
- 설정에서 즉시 전환
- AI 시스템 프롬프트 언어별 분기

## 명세 참조
- DIVE_SPEC.md §2.5 — 다국어 우선순위
- DIVE_SPEC.md §12.3 — 다국어 (i18n)

## 단계

1. `package.json` — `react-i18next`, `i18next`, `i18next-browser-languagedetector` 추가
2. `src/i18n/config.ts` — i18n 초기화, OS 언어 감지, fallback ko
3. `src/i18n/ko.json` / `src/i18n/en.json` — 키 구조:
   ```json
   {
     "app": { "title": "DIVE" },
     "sidebar": {
       "projects": "프로젝트",
       "newProject": "+ 새 프로젝트",
       "sessions": "세션"
     },
     "chat": {
       "placeholder": "메시지 입력...",
       "send": "보내기",
       "modelSelector": "모델 선택"
     },
     "workmap": { ... },
     "permission": {
       "approve": "승인",
       "modify": "수정",
       "reject": "거부"
     },
     "errors": {
       "network": "네트워크 연결이 불안정합니다. 잠시 후 다시 시도해주세요.",
       "auth": "API 키가 만료되었거나 유효하지 않습니다. 설정에서 재인증해주세요."
     }
     // ... 모든 사용자 노출 텍스트
   }
   ```
4. 모든 컴포넌트에서 `t('chat.placeholder')` 형식으로 호출:
   - 점진적 마이그레이션 — 작업 페이지마다 분리
   - 자동 감지 도구 (`i18next-parser` 등)로 미번역 키 발견
5. AI 시스템 프롬프트 (`src-tauri/src/agent/prompts/`):
   - `prompts_ko.rs` / `prompts_en.rs` 분리
   - 현재 사용자 언어를 Tauri command로 받아 적절한 prompt 사용
   - 한국어 모드 — "한국어로 응답하세요" 명시
   - 영어 모드 — "Respond in English" 명시
6. 변수 보간 — `{{count}}`, `{{filename}}` 같은 동적 값
7. 복수형 처리 (영어 위주) — `count_one`, `count_other`
8. 설정에서 언어 토글 즉시 반영 (i18n.changeLanguage)
9. 검증 — 모든 사용자 노출 텍스트가 i18n 리소스에 있는지 정적 분석:
   - `grep -rn "한국어 텍스트" src/` 후 미마이그레이션 발견 시 수정
10. 단위 테스트 — 두 언어 모두에서 키 존재 확인

## 완료 조건
- [ ] OS가 한국어면 자동 한국어, 영어면 자동 영어
- [ ] 설정에서 즉시 전환 가능
- [ ] 모든 UI 텍스트가 두 언어 모두 정상
- [ ] 미번역 키 0개 (정적 분석 통과)
- [ ] AI 시스템 프롬프트가 언어별로 분기됨
- [ ] AI 응답이 사용자 언어로 옴 (수동 확인)
- [ ] 호칭 없는 톤 유지 (영어도 "you" 사용 자제, 시스템 메시지 톤)
- [ ] 단위 테스트 통과

## 확인 질문
- 영어 톤 — 한국어 호칭 없음과 동등하게. "you"를 어떻게 피할지? "Click to..." 같은 명령형, "Project created" 같은 수동형 추천.
- 영어 폰트 — Inter / SF Pro Text. 명세 §2.3.
- 번역 품질 — 자동 번역 도구 vs 수동. 수동 추천 (전문 용어 일관성). 단 양 많아 시간 소요.
- 키 누락 시 — fallback ko 표시 vs 키 그대로. fallback 추천.
