# 작업 30: v0.3 통합 테스트

## 컨텍스트
Phase 5의 마지막. Codex OAuth (작업 25) + MCP (작업 26·27) + 프롬프트 도우미 (작업 28·29)가 모두 함께 동작하는지 통합 검증. 각 기능이 따로는 동작하지만 결합 시 충돌이 발생할 가능성을 사전에 발견.

## 이번 작업 범위
- Phase 5 통합 시나리오 작성·실행
- 4개 프로바이더 + MCP 도구 + 프롬프트 도우미를 조합한 end-to-end 테스트
- 회귀 테스트 — Phase 1~4 기능이 깨지지 않았는지
- 학술대회 발표 준비용 데모 시나리오 (선택)
- v0.3 릴리스 노트 초안

## 명세 참조
- DIVE_SPEC.md §3 — 사용자 시나리오
- DIVE_SPEC.md §13.7 — 10월 마일스톤

## 단계

1. **통합 시나리오 작성** (`docs/integration_scenarios/v03.md`):
   - 시나리오 1 — Codex OAuth로 ChatGPT 구독 사용자가 MCP 파일시스템 도구를 활용해 작업
   - 시나리오 2 — OpenRouter 자식 키 학생이 MCP 학습용 도구 (예: 채점 검증) 호출
   - 시나리오 3 — 프롬프트 도우미 + 보내기 전 점검을 결합한 신중한 사용자 흐름
   - 시나리오 4 — 4개 프로바이더 간 모델 전환하며 같은 카드 작업
2. **수동 통합 테스트 실행** — 각 시나리오를 실제로 따라가며 막히는 곳 기록
3. **회귀 테스트** — Phase 1~4 핵심 흐름 (시나리오 A·B) 다시 실행:
   - DIVE 4단계 게이트 정상 동작
   - 권한 카드 흐름 정상
   - 체크포인트 자동·수동 동작
   - 익명화 export 정상
4. **자동 통합 테스트** (선택, 가능하면):
   - Playwright 또는 Tauri 자체 e2e 도구로 시나리오 1·2 자동화
5. **버그 hotfix** — 발견된 문제 즉시 수정 (큰 변경은 별도 ADR + 다음 Phase 검토)
6. **v0.3 릴리스 노트** (`docs/releases/v0.3.md`):
   - 새 기능 — Codex OAuth, MCP, 프롬프트 도우미, 보내기 전 점검
   - 알려진 제약 — Codex 차단 위험, MCP OAuth 미지원
   - 마이그레이션 안내 (v0.2 → v0.3)
7. **학술대회 데모 시나리오** (선택):
   - 5분 이내 시연 가능한 짧은 흐름
   - DIVE 4단계 + 권한 카드 + MCP 통합을 보여줄 수 있는 예시

## 확인 질문
- 자동 통합 테스트 도입 — 시간 비용 vs 안정성 이익. Phase 6에서 본격 도입? 추천.
- Codex OAuth 차단 발생 시 — 회귀 테스트가 실패. 시나리오 1을 "Codex 사용 가능 시" 조건부로.
- 학술대회 데모 — 라이브 시연 vs 녹화. 녹화 안전. 단 라이브가 인상 강함. 둘 다 준비 추천.
- v0.3 vs v0.2 호환성 — SQLite 스키마 변경 있다면 마이그레이션 동작 검증 필수.
