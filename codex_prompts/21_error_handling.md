# 작업 21: 에러 처리 + 네트워크 재시도

## 컨텍스트
파일럿 환경에서 학교 네트워크 불안정·API 오류·도구 실패 등이 자주 발생할 수 있습니다. 명세 §9.6, §12.4의 에러 정책을 일관되게 적용합니다.

## 이번 작업 범위
- 네트워크 호출 자동 재시도 3회 (지수 백오프 0.5s, 1s, 2s)
- 3회 실패 시 명시적 에러 + 다른 프로바이더 전환 옵션
- OAuth 토큰 갱신 실패 자동 재인증 안내
- 도구 실행 실패 → AI에게 stderr 전달 → 재시도
- 동일 도구·인자 5회 연속 실패 시 무한 루프 방지
- 사용자 노출 에러 메시지 한국어, 호칭 없음, 친절

## 단계

1. `src-tauri/src/agent/retry.rs` — 재시도 정책:
   ```rust
   pub async fn with_retry<T, F>(f: F, max: usize) -> Result<T>
       where F: FnMut() -> BoxFuture<Result<T>>
   ```
   지수 백오프, 재시도 가능 에러만 (네트워크·rate_limit), 인증 실패는 재시도 X
2. 프로바이더 어댑터에 재시도 적용 — `LlmProvider::chat` 내부에서 자동 재시도
3. 에러 분류 — `ProviderError`:
   - `Network` (재시도 가능)
   - `RateLimit` (재시도 가능, longer backoff)
   - `Auth` (재인증 필요)
   - `InvalidRequest` (재시도 무의미)
   - `ServerError` (재시도 가능)
4. 3회 실패 시 — UI에 명시적 에러 토스트 + [다른 프로바이더로 전환] 버튼 + [재시도] 버튼
5. OAuth 토큰 갱신 실패 — Auth 에러 발생 시 토스트 "재인증 필요" + [설정 열기] 버튼
6. 도구 실행 실패 — stderr를 AI 메시지로 전달, AI가 자체 진단 후 재시도. UI에는 시스템 메시지로 알림.
7. 무한 루프 방지 — `src-tauri/src/agent/loop_guard.rs`:
   - 동일 (도구명, 인자 해시) 5회 연속 실패 추적
   - 한도 도달 시 강제 정지 + 사용자에게 안내
8. 에러 메시지 카탈로그 (`src/i18n/errors.ko.json`):
   - 친절한 톤, 호칭 없음
   - 예: "네트워크 연결이 불안정합니다. 잠시 후 다시 시도해주세요."
   - 예: "API 키가 만료되었거나 유효하지 않습니다. 설정에서 재인증해주세요."
9. 단위 테스트 — mock 에러로 재시도 동작·한도 검증

## 완료 조건
- [ ] 네트워크 일시 단절 시 자동 재시도 3회 후 사용자에게 명시 알림
- [ ] Rate limit 에러 시 더 긴 백오프 (예: 5초, 10초, 20초)
- [ ] Auth 에러 시 재시도 X, 재인증 안내
- [ ] 도구 실행 실패 시 AI에게 stderr 전달 + UI 시스템 메시지
- [ ] 동일 도구 5회 실패 시 강제 정지 + 사용자 안내
- [ ] 모든 에러 메시지 한국어, 호칭 없음
- [ ] 단위 테스트 통과

## 확인 질문
- 재시도 도중 사용자가 [중단] 가능 — `chat_cancel` 명령으로. 이미 작업 09에 있으면 재사용.
- Rate limit 백오프 — `Retry-After` 헤더 존중. OpenRouter는 이걸 보냄.
- 다른 프로바이더 전환 — 같은 메시지를 그대로 전송 vs 사용자 확인. 확인 추천.
- 무한 루프 정지 — 5회 vs 10회. 명세는 5회 명시. 그대로.
