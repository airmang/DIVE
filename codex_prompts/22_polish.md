# 작업 22: 빈 상태·로딩·토스트 폴리싱

## 컨텍스트
모든 화면의 빈 상태·로딩·토스트 일관성을 통일합니다. 파일럿에서 학생이 "지금 뭐가 일어나는지" 모르는 상황을 줄이는 것이 목표.

## 이번 작업 범위
- 빈 상태 일러스트 (프로젝트 없음, 카드 없음, 메시지 없음, 검색 결과 없음 등)
- 스켈레톤 로딩 — 메시지 스트리밍 시작 전, 카드 분해 중, 미리보기 빌드 중
- 토스트 시스템 통일 — 성공·정보·경고·에러 4종
- 마이크로 인터랙션 — 호버·포커스·트랜지션
- 에러 화면 — 치명적 에러 시 (DB 손상, 권한 문제 등) friendly fallback

## 명세 참조
- DIVE_SPEC.md §2.3 — 디자인 원칙 (그림자 없음, 8px 그리드, 200ms 트랜지션)

## 단계

1. `src/components/empty/` 빈 상태 컴포넌트들:
   - `EmptyProjects.tsx` — "+ 새 프로젝트로 시작하세요"
   - `EmptyCards.tsx` — "AI 도움 받기 또는 + 카드 추가로 시작하세요"
   - `EmptyMessages.tsx` — "첫 메시지를 입력해보세요"
   - `EmptySearch.tsx` — "검색 결과가 없습니다"
   - 각각 단순 일러스트 (CSS만으로 구현 가능한 단순 도형) + 안내 텍스트
2. `src/components/skeleton/` 스켈레톤:
   - 메시지 스켈레톤 (회색 바 3개)
   - 카드 스켈레톤
   - 미리보기 스켈레톤
3. 토스트 시스템 — `sonner` 또는 `react-hot-toast`:
   - 성공 (초록), 정보 (파랑), 경고 (주의), 에러 (위험)
   - 자동 사라짐 (성공 3초, 에러 5초, 경고는 사용자가 닫음)
   - 위치 — 우측 상단
4. 마이크로 인터랙션:
   - 모든 버튼 호버 시 `accent.hover` 적용 (200ms ease-out)
   - 카드 호버 시 미세한 elevation (border 색만 변경, 그림자 X)
   - 포커스 ring — `accent.default` 2px outline
   - 클릭 시 100ms 압축 효과
5. 에러 fallback — `src/components/ErrorBoundary.tsx`:
   - React Error Boundary
   - 치명적 에러 시 friendly 화면 ("문제가 발생했습니다. 앱을 다시 시작해주세요.")
   - [재시작] / [로그 복사] / [GitHub 이슈 보고] 버튼
6. 진행 상태 표시:
   - 카드 분해 중 — "AI가 카드를 제안하고 있습니다..." + 스피너
   - V 검증 중 — "AI가 자체 검증 중..." + 스피너
   - 체크포인트 생성 중 — 우측 상단 작은 아이콘
7. 모든 텍스트 한국어, 호칭 없음

## 완료 조건
- [ ] 모든 빈 상태에 일러스트 + 안내
- [ ] 스트리밍 시작 전 스켈레톤 표시
- [ ] 4종 토스트 일관된 스타일
- [ ] 호버·포커스·클릭 인터랙션 일관
- [ ] React Error Boundary 동작 (의도적 에러로 검증)
- [ ] 진행 상태 표시 — AI 분해·V 검증·체크포인트
- [ ] 모든 텍스트 호칭 없음

## 확인 질문
- 일러스트 — CSS 도형 vs SVG 일러스트. SVG 추천 (저렴하지만 인상 좋음). 무료 일러스트 라이브러리 (예: undraw.co) 사용 시 라이선스 확인.
- 토스트 라이브러리 — `sonner` 추천 (Vercel 디자인, 활발히 유지)
- 에러 fallback — 사용자 PC에서 자동으로 GitHub 이슈 생성? 아니면 단순 [복사] 버튼? 복사 추천 (개인정보 보호).
- 마이크로 인터랙션 과한지? — 200ms 이내 명세 준수, 그 이상은 X.
