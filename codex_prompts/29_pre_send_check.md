# 작업 29: 보내기 전 점검 + 템플릿 라이브러리 확장

## 컨텍스트
명세 §6.6.3의 "보내기 전 점검". AI가 사용자 프롬프트를 자체 비평. 모델 호출 1회 추가 소비하므로 사용자가 명시 호출.

## 이번 작업 범위
- 입력란 우측 작은 [점검] 버튼
- AI에게 자체 비평 요청 (시스템 프롬프트로 분리)
- 결과 — 모호한 부분, 보완 제안 표시
- 사용자가 적용 또는 무시 선택
- 토큰 사용량 표시

## 명세 참조
- DIVE_SPEC.md §6.6.3 — 보내기 전 점검

## 단계

1. `src/components/chat/PreSendCheck.tsx`:
   - 입력란 우측 작은 버튼 (✨ 옆에 또는 별도)
   - 호버 시 툴팁 "AI에게 프롬프트 점검 요청 (토큰 소비)"
   - 클릭 시 모달 또는 인라인 팝업
2. 백엔드 — `src-tauri/src/agent/critique.rs`:
   ```rust
   pub async fn critique_prompt(prompt: &str, current_card: &Card)
       -> Result<CritiqueResult>
   ```
3. 자체 비평 프롬프트 템플릿:
   ```
   다음은 사용자가 AI 코딩 에이전트에게 보내려는 프롬프트입니다. 다음 항목으로 평가하세요:
   1. 모호한 부분 (지시 대명사, 양사, 주어 생략 등)
   2. 빠진 정보 (입력·출력 예시, 제약 조건)
   3. 보완 제안

   현재 카드: {card.title}
   사용자 프롬프트: "{prompt}"

   JSON 형식으로 반환:
   {
     "ambiguities": [...],
     "missing_info": [...],
     "suggestions": [...],
     "rewritten_prompt": "..."
   }
   ```
4. 결과 표시 UI:
   - 모호한 부분 — 하이라이트 + 사유
   - 빠진 정보 — 체크리스트
   - 보완 제안 — 새 프롬프트 제안 + [이걸로 교체] 버튼
   - [무시하고 보내기] 버튼
5. 토큰 사용량:
   - 점검 호출 후 사용한 토큰 수 표시 ("점검에 1,234 토큰 사용")
   - 누적 토큰 사용량 — 설정 페이지에 표시
6. 빠른 점검 옵션 — 사용자가 점검 결과 보지 않고 [자동 적용] 모드 (rewritten_prompt를 자동 사용). 설정에서 옵트인.
7. 단위 테스트 — mock LLM 응답으로 결과 파싱·표시

## 확인 질문
- 점검 호출 모델 — 사용자가 선택한 모델 vs 항상 저렴한 모델 (예: haiku, gpt-5.1-mini). 저렴한 모델 추천 (학생 비용 부담 줄임). 단, 정확도 trade-off.
- 점검 결과 캐싱 — 같은 프롬프트 재점검 시 캐시. 입력 1글자 차이도 새 호출. 추천.
- 비평 톤 — 명세 §2.4 호칭 없음. AI가 "당신의 프롬프트는..." 같은 표현을 자제하도록 시스템 프롬프트에 명시.
- 자동 적용 모드 — 위험할 수 있음 (AI가 사용자 의도 왜곡 가능). 기본 OFF, 명시 옵트인.
