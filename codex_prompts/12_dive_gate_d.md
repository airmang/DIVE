# 작업 12: DIVE 게이트 D 단계 + 시나리오 A 통합 테스트

## 컨텍스트
v0.1의 마지막 작업. D 단계 게이트만 강제하고 (I·V·E는 placeholder), 시나리오 A의 1~6번 단계가 끝까지 동작하는지 통합 검증합니다. 이 작업이 끝나면 v0.1 데모가 가능해집니다.

## 이번 작업 범위
- DIVE Gate Engine (Rust) — D 단계만 강제
- "AI 도움 받기" 기능 — 사용자 요구를 받아 AI가 카드 초안 N개 제안
- 게이트 차단 시 사용자 가이드 메시지
- 시나리오 A 통합 시나리오 테스트
- I·V·E는 작업 13에서 강제

## 단계

1. `src-tauri/src/dive/gate.rs` — DIVE Gate Engine 골격:
   ```rust
   pub fn check_d_passed(workmap: &Workmap) -> Result<()>
   pub fn check_message_allowed(workmap: &Workmap, msg: &Message) -> Result<()>
   ```
2. D 단계 게이트 규칙:
   - 워크맵에 카드 0개 → 일반 채팅 메시지 차단
   - 단, "AI 도움 받기" 모드는 허용 (특수 메시지 타입)
   - 차단 시 `GateError::DStageNotPassed` 반환 + 가이드 메시지
3. Agent Loop에 gate 호출 추가 — `chat_send` 진입 시 gate.check_message_allowed
4. 프론트:
   - 게이트 차단 시 워크맵 위에 가이드 ("먼저 기능을 작은 단위로 분해하세요")
   - 채팅 입력란 disabled (D 통과 전)
   - "AI 도움 받기" 버튼 (워크맵 빈 상태에서 노출)
5. AI 도움 받기 흐름:
   - 사용자가 요구사항 입력 (예: "할 일 앱 만들고 싶어")
   - 시스템 프롬프트 — "이 요구를 작은 카드 단위로 분해해. 1~10개 사이로. JSON 형식으로 반환:"
   - AI가 카드 초안 JSON 반환
   - UI에 카드 4개 미리보기 + [모두 수락] / [개별 편집] 버튼
   - 수락 시 SQLite에 카드 저장, 워크맵 갱신
6. + 카드 추가 (수동) — 워크맵에서 직접 카드 추가 가능 (D 단계만 활성)
7. 통합 테스트 — 시나리오 A 1~6번:
   1. 빈 프로젝트 열기 → D 단계
   2. 채팅에 "할 일 앱 만들고 싶어" 입력 → 차단됨, 가이드 표시
   3. AI 도움 받기 → 카드 4개 제안 → 수락
   4. 첫 카드 클릭 → I 단계 진입 (이번 작업은 I 게이트 X, placeholder)
   5. 채팅 입력 가능 → AI가 edit_file 호출 → 권한 카드 → 승인
   6. 슬라이드 인 패널의 미리보기 탭에서 결과 확인

## 확인 질문
- AI 도움 받기 — 카드 N개를 한 번에 받을지(`response_format: json`), 점진적으로 받을지? 한 번에 추천 (사용자 검토 용이)
- 카드 분해가 너무 많거나 너무 적을 때 — AI에게 1~10개 가이드 줘도 가끔 11개 이상 나옴. 자동 합치기? 거절? 거절 추천 (사용자 명확화 유도)
- 차단 메시지 톤 — 명세 §2.4 호칭 없음 원칙. "먼저 기능을 작은 단위로 분해하세요" 같은 시스템 톤
- `--features no-gate` 빌드 플래그 — 명세 §4.7. 이번 작업에서 추가? 또는 v0.2 마무리에서? 이번 작업에 미리 넣어두는 게 안전.
