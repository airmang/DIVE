# 작업 20: 설정 화면 (프로바이더·자동 승인·테마)

## 컨텍스트
사용자가 시스템을 자유롭게 변경할 수 있는 메인 진입점. 프로바이더 관리 + 자동 승인 정책 + 테마 + 언어 + 데이터 내보내기 진입점.

## 이번 작업 범위
- 설정 화면 라우팅 — `Ctrl+,` 또는 사이드바 하단 톱니바퀴
- 탭 구성 — 프로바이더 / 권한 / 테마 / 데이터 / 정보
- 프로바이더 카드 5종 (Anthropic / OpenAI / Codex OAuth placeholder / OpenRouter / Custom)
- 자동 승인 정책 토글 (안전 도구만)
- 테마 토글 (다크/라이트/시스템)
- 데이터 내보내기 진입점 (작업 18 ExportDialog 호출)

## 단계

1. `src/pages/SettingsPage.tsx` — 모달 또는 별도 라우트
2. 좌측 탭 — Providers / Permissions / Theme / Data / About
3. **Providers 탭**:
   - 5종 프로바이더 카드 — 이름 + tier 뱃지 (기본/부수) + 인증 방식 + 모델 목록 + 연결 상태 점
   - 미연결 — [연결하기] 버튼
   - 연결됨 — [재인증] / [✕ 연결 해제] 버튼, 마지막 사용 시각
   - OpenRouter Provisioning Keys 섹션 (작업 17 재사용) — 메인 키 등록 시 활성
4. **Permissions 탭**:
   - 도구별 자동 승인 토글 — 안전 도구 (`read_file`, `list_dir`, `search_files`)만 활성
   - 주의·위험 도구의 토글은 회색 disabled
   - 신뢰하는 디렉터리 화이트리스트 (옵션) — 명세 §3.2 시나리오 B
   - 차단 명령 목록 표시 (읽기 전용) — 사용자 인지 위해
5. **Theme 탭**:
   - 다크 / 라이트 / 시스템 자동 라디오 버튼
   - 언어 토글 — 한국어 / English (작업 31에서 실제 i18n 적용)
6. **Data 탭**:
   - [데이터 내보내기] 버튼 → ExportDialog 호출
   - DB 위치 표시 (`%APPDATA%\dive\` 또는 프로젝트별 `.dive/`)
   - [모든 데이터 삭제] 버튼 (확인 모달, 위험 표시)
7. **About 탭**:
   - DIVE 버전, 라이선스, GitHub 링크
   - 의존 라이브러리 라이선스 표시

## 확인 질문
- 설정 화면 — 모달 vs 별도 페이지. 별도 페이지 추천 (라우팅 일관성, 작업 영역 넓음)
- 자동 승인 정책 변경 — 즉시 반영 vs 다음 차시. 즉시 추천 (사용자 의도 명확)
- 신뢰하는 디렉터리 화이트리스트 — 이번 작업에 포함 vs 후속 작업. 이번에 placeholder만 (UI), 실제 동작은 후속.
- "모든 데이터 삭제" — 정말 무서운 동작. 추가 확인 ("DELETE 입력") 권장.
