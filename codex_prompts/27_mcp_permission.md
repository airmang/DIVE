# 작업 27: MCP 도구 권한 카드 통합

## 컨텍스트
작업 26에서 MCP 클라이언트 기반을 만들었으니, 이제 MCP 도구도 내장 도구와 동일한 권한 카드 흐름을 거치도록 통합합니다. 위험도 분류, diff 표시(쓰기 도구의 경우), 승인/수정/거부.

## 이번 작업 범위
- MCP 도구 호출 시 Permission Hook 거치기
- 위험도별 권한 카드 표시 (안전·주의·위험)
- MCP 도구 입력·출력 정보 카드에 노출
- 사용자가 인자 수정 후 실행 가능
- 자동 승인 정책 적용 (안전 도구만 옵트인)

## 명세 참조
- DIVE_SPEC.md §8.5 — MCP 통합 (도구 위험도)
- DIVE_SPEC.md §5.5 — 권한 카드
- DIVE_SPEC.md §6.4 — 권한 시스템

## 단계

1. Permission Hook 확장 (작업 10에서 만든 것):
   - MCP 도구 식별 (`name` prefix `mcp::`)
   - 위험도 결정:
     1. 사용자 명시 설정 (작업 26 UI)
     2. MCP 서버 메타 힌트
     3. 기본값 (주의)
2. MCP 도구용 권한 카드 컴포넌트:
   - 기본 구조는 내장 도구와 동일
   - 추가 표시 — 어느 MCP 서버인지 ("server: filesystem-mcp")
   - 입력 인자 — JSON pretty-print (작업 10의 DiffViewer 일부 재사용)
   - 출력 결과 — 도구 실행 후 표시 (텍스트, 이미지, 구조화 데이터 등)
3. MCP 도구 출력 형식 다양성 처리:
   - 텍스트 — 그대로 표시
   - 이미지 — base64 → img 태그
   - 구조화 데이터 — JSON pretty-print 또는 표 (간단한 경우)
   - 에러 — 빨간 박스
4. 사용자 인자 수정:
   - 권한 카드의 [수정] 버튼 → JSON 편집 가능 인라인 에디터
   - 검증 — MCP 도구의 input_schema와 일치하는지 (작업 26에서 schema fetch)
5. 자동 승인 정책 — 명세 §8.3:
   - 사용자가 MCP 도구를 안전으로 분류한 경우만 자동 승인 가능
   - 주의·위험은 항상 수동
6. 차단 정책 — 명세 §9.2 블록리스트는 MCP 도구에는 적용 X (MCP는 사용자가 명시 등록한 신뢰 도구 가정). 단, MCP 도구가 bash 같은 위험 도구면 사용자가 위험 분류해야 함.
7. 단위 테스트 — mock MCP 도구로 권한 카드 흐름 검증

## 확인 질문
- JSON 인자 편집 — 자유 편집 vs schema 기반 폼. 자유 편집 + 검증 추천 (구현 단순, 고급 사용자 친화).
- 이미지 출력 — 권한 카드 안에서? 또는 슬라이드 인 패널? 권한 카드 작은 미리보기 + 클릭 시 슬라이드 인 패널 추천.
- 에러 출력 — 사용자에게도 노출? AI에게도 전달? 둘 다. AI가 다음 시도 결정 가능.
- 위험도 시각 표시 — 카드에 MCP 서버 이름과 함께 위험도 뱃지. 사용자가 한눈에 식별.
