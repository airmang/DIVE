# 작업 28: 프롬프트 도우미 + 모호함 감지

## 컨텍스트
입문자 학습 지원 핵심 기능. DIVE 단계별 템플릿 + 실시간 모호함 감지. 명세 §6.6.

## 이번 작업 범위
- 단계별 템플릿 라이브러리
- 사이드 패널 — 입력란 ✨ 버튼 클릭 시
- 실시간 모호함 감지 (debounced 500ms, 정규식+휴리스틱)
- 모호한 표현 하이라이트 + 힌트 (외부 호출 없음, 로컬 분석)

## 단계

1. 템플릿 라이브러리 (`src/lib/prompt_templates.ts`):
   ```ts
   const TEMPLATES = {
     D: [
       { title: "기능 분해", body: "[기능]을 만들고 싶어. 작은 단위로 분해해줘." },
       { title: "큰 그림", body: "이 프로젝트의 큰 그림을 먼저 잡아줘." },
       // ...
     ],
     I: [
       { title: "범위 한정", body: "[현재 카드]만 작업해줘. 다른 부분은 건드리지 마." },
       { title: "입출력 명시", body: "이 부분의 입력과 출력 예시를 먼저 알려줘." },
     ],
     V: [
       { title: "검증 방법", body: "이 카드의 동작을 어떻게 검증할 수 있어?" },
       { title: "테스트 케이스", body: "테스트 케이스 3개로 확인해줘." },
     ],
     E: [
       { title: "통합 점검", body: "전체 코드를 검토하고 통합 시 문제될 부분을 알려줘." },
     ],
   }
   ```
2. `src/components/chat/PromptHelper.tsx`:
   - 입력란 ✨ 버튼 클릭 시 우측 사이드 패널 (또는 popover)
   - 현재 DIVE 단계에 맞는 템플릿 카드 표시
   - 카드 클릭 시 입력란에 자동 삽입 (`[기능]` 같은 변수는 하이라이트)
3. `src/lib/ambiguity_detector.ts`:
   - debounced 500ms (lodash.debounce 또는 자체)
   - 패턴:
     - 지시 대명사: "이거", "그것", "저번 거", "그", "이"
     - 모호한 양사: "조금", "많이", "적당히", "약간"
     - 주어 생략: 동사로 시작하는 명령문 ("만들어줘", "수정해줘")
     - 단수/복수 모호: "버튼들" + 개수 미명시
   - 정규식 + 휴리스틱 조합. 외부 LLM 호출 없음 (개인정보 보호).
4. UI:
   - 입력 중 모호한 표현이 감지되면 해당 단어 하이라이트 (밑줄 또는 배경색)
   - 입력란 우측에 작은 경고 아이콘 (⚠) — 호버 시 툴팁 ("어떤 버튼인지 명시해주세요")
   - 입력 중에만 표시, 전송 후에는 사라짐
5. 사용자가 무시하고 전송 가능 — 강제 차단 X (학습 보조 도구일 뿐)
6. 다국어 — 한국어 패턴 우선. 영어 버전은 작업 31에서.
7. 단위 테스트 — 다양한 모호 케이스 vs 명확 케이스로 정확도 측정

## 완료 조건
- [ ] ✨ 버튼 클릭 시 사이드 패널 노출
- [ ] DIVE 단계 변경에 따라 템플릿 카드 자동 갱신
- [ ] 템플릿 클릭 시 입력란에 자동 삽입
- [ ] 입력 중 모호한 표현 자동 하이라이트 (debounced 500ms)
- [ ] 호버 시 친절한 힌트
- [ ] 외부 LLM 호출 없음 (네트워크 탭으로 확인)
- [ ] 단위 테스트 통과 (정확도 70% 이상)

## 확인 질문
- 정규식 정확도 한계 — 100% 잡지 못함. 거짓 양성 (정상 표현을 모호로 분류) 줄이는 게 우선. 거짓 양성 < 5% 목표.
- 사이드 패널 위치 — 입력란 위 popover vs 별도 패널. popover 추천 (작업 흐름 끊지 않음).
- 사용자가 끄기 가능 — 설정에서 모호함 감지 토글. 추천.
- 학생 활동지에 템플릿 인쇄 — 작업 24에서 이미 다룸. 코드와 일관성 유지.
