# 작업 24: 사용자·교사 매뉴얼 + 차시별 시나리오 테스트

## 컨텍스트
파일럿 운영 자료. 학생용 빠른 시작, 교사용 운영 매뉴얼, 6차시 시나리오 워크스루를 만들고 코드 흐름과 일치하는지 사용자 시나리오로 검증.

- 교사용 운영 매뉴얼 (10~15페이지)

- 시나리오 워크스루 — 명세 §3.1, §3.2를 끝까지 따라가며 동작 확인

## 단계

1. **학생용 빠른 시작** (`docs/student_quickstart.md`):
   - 첫 실행 — 수업용 키 등록 (QR)
   - DIVE 4단계 한 줄 설명 (분해 → 지시 → 검증 → 확장)
   - 첫 프로젝트 만들기
   - 권한 카드 — 안전·주의·위험 의미
   - 막힐 때 도움 요청 방법
2. **교사용 운영 매뉴얼** (`docs/teacher_manual.md`):
   - 차시 전 준비 — 메인 키 등록, 자식 키 발급, 학생 PC 사전 설치
   - 차시 진행 — 학생이 막힌 패턴 (분해 못 함, 권한 카드 무조건 승인 등) 대응
   - 차시 후 — 자식 키 폐기, export 생성, 분석 자료 추출
   - 트러블슈팅 — 흔한 문제와 해결책
   - 차시별 학습 목표
3. **6차시 지도안** (`docs/lesson_plans/`):
   - 차시 1 — DIVE 소개 + 첫 프로젝트
   - 차시 2 — D 단계 분해 연습
   - 차시 3 — I 단계 명확한 지시
   - 차시 4 — V 단계 검증 + 권한 카드
   - 차시 5 — 통합 + 확장
   - 차시 6 — 발표 + 회고
   - 각 차시마다 학습 목표·활동·산출물·평가 기준
4. **학생 활동지** — 차시별 1장씩 (`docs/worksheets/`)
5. **시나리오 워크스루** (수동 검증):
   - 시나리오 A 1~11번 단계 따라가기 — 막히는 곳 없는지
   - 시나리오 B 1~7번 단계 따라가기 — 익숙한 사용자 흐름
   - 발견된 UX 문제는 별도 이슈 등록 (이번 작업에서 코드 수정 X, Phase 4 후 hotfix)
6. 모든 문서 한국어, 호칭 없음

## 확인 질문
- 매뉴얼 형식 — Markdown vs PDF. Markdown 추천 (수정 용이), PDF는 자동 변환 (`pandoc`).
- 지도안 — 한국 학교 지도안 양식 (도입·전개·정리·평가) 따를지? 김나영·양회욱 교사 협력 결정.
- 차시 1 시작점 — DIVE 4단계 강의 후 실습 vs 바로 실습. 후자 추천 (게이트 차단 가이드가 자연스러운 학습 지점).
- 학생 활동지 — 출력 인쇄용 vs DIVE 안 어디 통합? 인쇄 추천 (학습 환경에서 디지털 + 종이 병행).
